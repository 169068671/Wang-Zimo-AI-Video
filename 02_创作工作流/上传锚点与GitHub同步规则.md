# 上传锚点与 GitHub 同步规则

> [!important] 双层锚点制度
> 原始高分辨率锚点是本地生产母版；上传锚点是同一人物、场景或品牌锚点的轻量派生副本。两者锚定含义一致，但用途不同，上传锚点不得反向覆盖原始母版。

## 强制规则

1. 本地图片生成、精修、身份、场景与品牌核验，以及成片验收，只使用原始高分辨率锚点。
2. GitHub 上传、在线预览、Markdown 展示，以及 **IMA / MiniMax / 可灵 / OpenRouter 等 API 参考图上传**，统一使用 `attachments/上传锚点/` 中的一一对应小图；**禁止上传原始 4K/高分辨率大文件**。
3. 上传锚点最长边为 1600 像素、JPEG，单张不得超过 2 MB。
4. 原图保留在本地原目录，不覆盖、不删除；同步插件必须排除原始锚点目录。
5. 每张小图必须在 `attachments/上传锚点/上传锚点清单.json` 保存原图路径、原图哈希、小图路径、小图哈希、尺寸与体积。
6. 原始锚点更新后，必须重新生成对应上传锚点；哈希不一致时禁止上传。
7. `08_原始项目副本` 不是正式上传锚点池，不从其中重复派生上传图。
8. 本地生成、精修与身份/场景验收用原目录大图；凡需上传到云端的参考图，一律改选 `attachments/上传锚点/` 对应小图。

## GitHub 同步与 Git LFS

Obsidian「一键同步」插件（`*-github-sync`）会自动检测 `.gitattributes` 并启用本地 Git LFS；单次同步超时约 **30 分钟**，以支持音频/视频大文件推送。

| 内容 | 同步方式 |
|---|---|
| 音频 / 视频 / 普通成片图片（如 `attachments/声音资产/`、成片等） | **Git LFS** |
| `attachments/上传锚点/**` 小图（≤2 MB） | **普通 Git** |
| `attachments/人物锚点`、`场景锚点`、`品牌锚点`、`4K锚点组` 等原图母版 | **Git LFS**（与成片一样上传） |

本机需已安装：`brew install git-lfs && git lfs install`。原图母版与音频/视频一并经 Git LFS 推送。
